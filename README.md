## Overview

This project builds a **binary classification pipeline** to predict diabetes outcomes. It is structured around reproducible ML principles:

- **Data versioning** with [DVC](https://dvc.org/) backed by remote storage (S3)
- **Experiment tracking** with [MLflow](https://mlflow.org/) hosted on [DagsHub](https://dagshub.com/)
- **Hyperparameter tuning** via `GridSearchCV`
- **Model serialization** with `pickle` and MLflow model registry
- **Configurable parameters** centralized in `params.yaml`

## 🚦 Getting Started

### Prerequisites

- Python **3.8+**
- `pip` or a virtual environment manager (e.g., `venv`, `conda`)
- An [AWS account](https://aws.amazon.com/) (for DVC S3 remote, optional for local runs)
- A [DagsHub account](https://dagshub.com/) (for MLflow tracking)

---

### Installation

1. **Clone the repository**

   ```bash
   git clone https://dagshub.com/Sai-Bhargava-S/MLpipeline.git
   cd MLpipeline
   ```

2. **Create and activate a virtual environment**

   ```bash
   # Windows
   python -m venv mlenv
   mlenv\Scripts\activate

   # macOS / Linux
   python -m venv mlenv
   source mlenv/bin/activate
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

---

### Setting Up DagsHub & MLflow

This project uses **DagsHub** as the remote MLflow tracking server. Set the following environment variables before running any pipeline stage:

```bash
# Windows (PowerShell)
$env:MLFLOW_TRACKING_URI      = "https://dagshub.com/<your-username>/MLpipeline.mlflow"
$env:MLFLOW_TRACKING_USERNAME = "<your-dagshub-username>"
$env:MLFLOW_TRACKING_PASSWORD = "<your-dagshub-token>"

# macOS / Linux
export MLFLOW_TRACKING_URI="https://dagshub.com/<your-username>/MLpipeline.mlflow"
export MLFLOW_TRACKING_USERNAME="<your-dagshub-username>"
export MLFLOW_TRACKING_PASSWORD="<your-dagshub-token>"
```

> **Tip:** You can find your DagsHub token at **Settings → Access Tokens** on your DagsHub profile.

## ⚙️ Pipeline Stages

### 1. Preprocessing

**Script:** `src/preprocess.py`

Reads the raw CSV, applies cleaning/transformations, and saves the processed dataset.

```bash
python src/preprocess.py
```

**Parameters (from `params.yaml`):**

```yaml
preprocess:
  input:  data/raw/data.csv
  output: data/processed/data.csv
```

---

### 2. Training

**Script:** `src/train.py`

- Loads the raw data and splits it into train/test sets (80/20)
- Performs **hyperparameter tuning** using `GridSearchCV` (3-fold CV) over:

  | Parameter | Values |
  |-----------|--------|
  | `n_estimators` | 100, 200 |
  | `max_depth` | 5, 10, None |
  | `min_samples_split` | 2, 5 |
  | `min_samples_leaf` | 1, 2 |

- Trains the best **Random Forest Classifier** found by the grid search
- Logs the following to MLflow:
  - `accuracy` metric
  - Best hyperparameters
  - Confusion matrix (as artifact text)
  - Classification report (as artifact text)
  - Trained model (registered as `Best Model` in the MLflow registry)
- Saves the model locally as `models/model.pkl`

```bash
python src/train.py
```

**Parameters (from `params.yaml`):**

```yaml
train:
  data:          data/raw/data.csv
  model:         models/model.pkl
  random_state:  42
  n_estimators:  100
  max_depth:     5
```

---

### 3. Evaluation

**Script:** `src/evaluate.py`

- Loads the saved `model.pkl` from disk
- Runs inference on the full dataset
- Logs the `accuracy` metric back to MLflow

```bash
python src/evaluate.py
```

## 🔧 Configuration

All pipeline parameters live in a single file — **`params.yaml`** — making it easy to modify settings without touching source code:

```yaml
preprocess:
  input:  data/raw/data.csv
  output: data/processed/data.csv

train:
  data:          data/raw/data.csv
  model:         models/model.pkl
  random_state:  42
  n_estimators:  100
  max_depth:     5
```

## ▶️ Running the Pipeline

You can run each stage individually or use DVC to run the full pipeline end-to-end (if a `dvc.yaml` is configured):

**Run stages individually:**

```bash
python src/preprocess.py
python src/train.py
python src/evaluate.py
```

**Run with DVC (if `dvc.yaml` is set up):**

```bash
dvc repro
```

DVC will automatically detect which stages are out of date (based on changed inputs or params) and re-run only those stages.

## 🚢 Model Deployment

The best model is automatically registered in the **MLflow Model Registry** under the name **`Best Model`** after each training run. From the registry you can:

1. **Promote to Staging/Production** via the DagsHub/MLflow UI
2. **Load and serve the model** using MLflow's built-in serving:

```bash
# Serve the registered model via a REST API
mlflow models serve \
  -m "models:/Best Model/Production" \
  --port 5001
```

3. **Load the model locally** for batch inference:

```python
import mlflow.sklearn

model = mlflow.sklearn.load_model("models:/Best Model/Production")
predictions = model.predict(X_new)
```

4. **Use the pickled model** directly (offline):

```python
import pickle
import pandas as pd

model = pickle.load(open("models/model.pkl", "rb"))
data  = pd.read_csv("data/raw/data.csv").drop(columns=["Outcome"])
preds = model.predict(data)
```
